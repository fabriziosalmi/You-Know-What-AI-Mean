# Security

This repository contains documentation only. There is no executable code, no service, no user data.

## For issues in the linked tools

The artifacts referenced in *What came next* are separate repositories with their own security policies. For a security issue in any of them, use that repository's reporting channel directly. If a referenced repository lacks an explicit security policy, default to a private message to the maintainer rather than opening a public issue.

## For factual errors in this guide

Factual errors (misattributed framework, broken link, misquoted citation) are not security issues. Open a regular issue.
